# Internationalization (i18n) (Tier 2)

Codify keeping user-facing text and locale formatting out of code logic. Balance policy applies: match the repo's existing catalog tooling and key convention. Do not introduce a second i18n system alongside an established one.

## Decisions worth encoding

- **No hardcoded user-facing strings.** Route every label, message, email, and error shown to users through the translation layer (Laravel `__()`/`trans()` + `lang/`, `vue-i18n` `t()`, gettext). Internal logs/exceptions stay in the source language — do not translate those.
- **Key naming.** Use namespaced, stable keys (`auth.login.failed`, `cart.item.removed`), not the English sentence as the key — UNLESS the repo already uses natural-language keys consistently, then follow it. Keys describe intent, not wording, so copy edits do not churn keys.
- **Pluralization via the framework.** Use built-in plural rules (`trans_choice()`, ICU `{count, plural, ...}`, `$t` with plural). Never hand-roll `if (n === 1)` — it breaks for locales with more than two plural forms.
- **Locale-aware formatting.** Dates, numbers, and currency go through `Intl` (`Intl.DateTimeFormat`, `Intl.NumberFormat`), localized Carbon, or the framework helper — never `sprintf`/manual `d/m/Y` string building.
- **Externalize per-locale content.** One catalog per locale; no locale branching in components.
- **Fallback locale.** Configure a fallback so a missing key degrades gracefully instead of rendering the raw key.
- **RTL awareness.** Use logical CSS (`margin-inline-start`, `text-align: start`) and set `dir` from locale; avoid hardcoded left/right (see `../packages/frontend/tailwind.md`).
- **Keep translations out of logic.** No business branching on translated text; branch on locale codes or keys.

## Example

❌ Bad — hardcoded string, manual date, hand-rolled plural:

```js
const label = count === 1 ? "1 item" : `${count} items`;
const when  = `${d.getDate()}/${d.getMonth() + 1}/${d.getFullYear()}`; // wrong in many locales
return `${label} added on ${when}`;
```

✅ Good — translation key with plural, locale-aware date:

```js
// Catalog owns wording + plural rules; Intl owns formatting.
const label = t('cart.items', count);            // "cart.items": "{count} item | {count} items"
const when  = new Intl.DateTimeFormat(locale).format(d);
return t('cart.added_on', { label, when });
```

Source: best-practice · Confidence: high
